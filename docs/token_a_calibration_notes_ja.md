# Token A 校正メモ（Phase1）

このメモは、`Token A` の日次配布額をどう読むべきか、そして将来どのように現実の生活費感覚へ寄せるかを整理するための補助資料です。

## 現在の意味

- 現在の `dailyUBIAmount` の初期値は `100 Token A / 日`。
- これは **Phase1 の制度挙動を確認するための初期パラメータ** であり、現実の最低生活費に校正済みの値ではない。
- 発生単位は日次だが、請求は週次・月次などのまとめ取りでもよい。

## なぜ現実価格にまだ合わせていないか

Phase1 の優先順位は、次の確認にある。

- UBI と減価が短期で破綻しないこと
- Treasury と Covenant の会計が壊れないこと
- dispute が弱者に一方的に不利になりすぎないこと
- participant が UI と主要フローを理解できること

したがって、今は **制度の形を先に検証し、絶対額の厳密な生活費校正は次段階で行う** 方針を採る。

## 将来の校正方針

将来的には、`Token A` を次の観点で調整する。

1. `APPI`
   内部購買力インデックスとして、最低生活バスケットの変動を日次配布額へ反映する。
2. `Survival Buffer`
   減価の床として、完全な消滅ではなく最低限の生存余地を残す。
3. `Village-specific baseline`
   村ごとに地域差や制度差を持てるよう、初期 `dailyUBIAmount` を parameter 化する。

## 現実の生活費に寄せるための次段階

Phase1 の次に必要なのは、以下のような基準作りである。

- 最低食費バスケット
- 最低住居負担の proxy
- 通信・移動コストの最低ライン
- 地域差を反映した APPI baseline

## 要点

- `100 Token A / 日` は本番の生活費保証額ではない。
- 現在は「制度が壊れず回るか」を見るための初期値である。
- 本格的な物価校正は `APPI` と `Survival Buffer` を使って次段階で行う。
